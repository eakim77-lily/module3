# be-test-code
개발한 코드에 대한 테스트 코드를 작성합니다.

## 세부내용
- 테스트 코드는 backend/test/ 아래에 작성합니다.
- 테스트 코드를 작성하고, test를 진행합니다.
  - 테스트 결과는 `/.cluade/docs/be-test.md 에 작성합니다.
  - 테스트 결과를 []에 체크박스로 표현합니다.

# be-debugging
디버깅 진행 내역을 로깅합니다.

## 세부내용
- 디버깅을 통한 전후 결과를 기록합니다.



# be-CRUD
개발 코드에 대한 CRUD 테스트 코드를 작성합니다.

# 세부내용
- backend 코드 중에 CRUD 관련 기능에 대한 테스트 코드를 작성합니다.
- 테스트 코드는 backend/CRUD/ 아래에 작성합니다.
- BE/도메인/테스트 식으로 작성하고, feature 넘버를 작성합니다.
- 테스트 결과를 []에 체크박스로 표현합니다.


# be-refactoring
개발해 놓은 코드에서 공용 모듈을 추출하여, 중복을 제거합니다. 

# 세부내용
- 추출한 공용모듈은 backend/refactoring/ 아래에 작성합니다.
- 공용모듈로 대체할 수 있는 부분들을 표시해줍니다.
